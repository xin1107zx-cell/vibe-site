# Vibe Site - MVP 需求文档

## 项目概述

**项目名称：** Vibe Site  
**项目定位：** 个性化占卜测试平台  
**核心价值：** 通过出生日期、血型、塔罗牌等维度，为用户提供个性化的人生指引  
**目标用户：** 18-35岁，对占卜、心理测试感兴趣的年轻人  

---

## MVP 核心功能

### 1. 用户系统（简化版）

**1.1 匿名访问**
- 用户无需注册即可进行测试
- 使用浏览器 LocalStorage 存储临时数据
- 生成唯一设备 ID 用于历史记录关联

**1.2 可选注册/登录**
- 邮箱 + 密码注册
- 登录后可跨设备查看历史记录
- 暂不支持第三方登录（后续迭代）

---

### 2. 测试流程

**2.1 信息收集页面**

输入项：
- 出生日期（年/月/日）
- 血型（A/B/O/AB/不确定）
- 测试类型选择（爱情/友情/事业）

**2.2 塔罗牌抽牌**

交互方式：
- 展示 22 张大阿卡纳牌背面
- 用户点击选择 3 张牌
- 翻牌动画展示结果

**2.3 结果生成**

生成逻辑：
- 根据星座（出生日期）+ 血型 + 塔罗牌组合
- 从预设文案库中匹配内容
- 生成个性化报告（800-1200 字）

报告结构：
- 标题：根据测试类型生成
- 核心解读（正向鼓励为主）
- 行动建议（3-5 条）
- 毒舌彩蛋（10% 概率触发）

---

### 3. 历史记录

**3.1 记录内容**
- 测试时间
- 测试类型
- 抽取的塔罗牌
- 生成的报告

**3.2 查看方式**
- 时间线展示
- 支持筛选（按类型）
- 点击查看完整报告

---

### 4. 多语言支持

**4.1 语言切换**
- 中文（简体）
- English
- 右上角切换按钮

**4.2 翻译范围**
- UI 文案
- 塔罗牌名称
- 报告内容（预设双语文案）

---

## 数据库设计

### 用户表（users）

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE,
  password_hash VARCHAR(255),
  device_id VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 测试记录表（readings）

```sql
CREATE TABLE readings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  device_id VARCHAR(255),
  birth_date DATE,
  blood_type VARCHAR(10),
  test_type VARCHAR(50),
  tarot_cards JSONB,
  report_content TEXT,
  language VARCHAR(10),
  created_at TIMESTAMP DEFAULT NOW()
);
```

## API 设计

### 用户相关

```
POST /api/auth/register    # 注册
POST /api/auth/login       # 登录
GET  /api/auth/me          # 获取当前用户信息
```

### 测试相关

```
POST /api/readings         # 创建测试记录
GET  /api/readings         # 获取历史记录
GET  /api/readings/:id     # 获取单条记录
```

### 数据相关

```
GET  /api/tarot/cards      # 获取塔罗牌列表
POST /api/tarot/draw       # 抽牌（返回随机 3 张）
POST /api/report/generate  # 生成报告
```

## UI/UX 设计要求

### 设计风格

- **简约未来感**：深色背景 + 渐变色点缀
- **流畅动画**：翻牌、页面切换使用平滑过渡
- **响应式设计**：适配桌面端和移动端

### 色彩方案

```
主色调：深蓝/紫色渐变 (#1a1a2e → #16213e)
强调色：金色/青色 (#ffd700 / #00d4ff)
文字色：白色/浅灰 (#ffffff / #e0e0e0)
```

### 关键页面

1. **首页**：简洁介绍 + "开始测试" 按钮
2. **信息收集页**：表单输入，进度条显示
3. **抽牌页**：22 张牌展示，翻牌动画
4. **结果页**：报告展示，分享按钮
5. **历史记录页**：时间线展示

## 内容策略

### 塔罗牌数据

- 使用 22 张大阿卡纳牌
- 每张牌包含：
  - 牌名（中英文）
  - 正位/逆位含义
  - 关键词（3-5 个）

### 报告文案库

**结构：**
```
星座（12种）× 血型（4种）× 测试类型（3种）× 塔罗组合
= 预设 144 个基础模板
```

**风格：**
- 70% 正向鼓励
- 20% 中性分析
- 10% 毒舌彩蛋

## 成功指标

### MVP 阶段目标

- 完成率：80% 用户完成完整测试流程
- 留存率：30% 用户进行第二次测试
- 分享率：20% 用户分享结果到社交媒体

### 技术指标

- 页面加载时间 < 2 秒
- API 响应时间 < 500ms
- 移动端适配完美

## 风险与应对

### 技术风险

- **数据库性能**：初期使用免费额度，需监控查询性能
- **API 限流**：设置合理的请求频率限制

### 内容风险

- **文案质量**：需要专业占卜师审核内容
- **用户隐私**：明确隐私政策，不存储敏感信息

### 运营风险

- **用户增长**：初期依赖社交媒体推广
- **成本控制**：使用免费/低成本服务，控制初期投入
